Procedimientos Almacenados

Los procedimientos almacenados son conjuntos de instrucciones SQL que se almacenan en la base de datos y pueden ser invocados por aplicaciones o usuarios. A diferencia de las funciones, los procedimientos pueden realizar múltiples operaciones y no necesariamente devuelven un valor.

Ejemplos de Procedimientos Almacenados

-- Para una venta de libros, creamos un procedimiento almacenado que retorna 
todos los libros de los cuales hay menos de 10 disponibles
delimiter //
 create procedure pa_libros_limite_stock()
 begin
   select * from libros
   where stock<=10;
 end //
 delimiter ;

Para llamar luego al procedimiento almacenado debemos utilizar la cláusula 'call':

call pa_libros_limite_stock();

Los procedimientos almacenados son fundamentales en el desarrollo de aplicaciones empresariales, ya que permiten implementar lógica de negocio compleja directamente en la base de datos, mejorando la seguridad, el rendimiento y la mantenibilidad.
